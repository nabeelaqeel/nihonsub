**Tag:** `v0.1`

## Summary

Initial MVP of NihonSub. File transcription and Linux live listening both functional.

## Added

### File Transcribe
Batch transcribe video/audio files to Japanese subtitles.

- Input formats: MP4, MKV, MP3, WAV, M4A (via ffmpeg)
- Pipeline: ffmpeg extraction → 16kHz mono WAV → silero-vad speech segmentation → faster-whisper ASR → .srt / .vtt output
- Model support: tiny, base, small, medium, large-v3 (default: medium)
- VAD configuration: threshold, min speech/silence duration via `.env`
- CLI: `python -m src transcribe <input> [-o output] [--model_size] [--subtitle_format]`

Key files:
- `src/audio/extractor.py` — ffmpeg audio extraction
- `src/audio/processor.py` — VAD + resampling for file mode
- `src/subtitle/generator.py` — .srt / .vtt generation
- `src/transcription/engine.py` — faster-whisper wrapper

### Live Listen (Linux only)
Real-time system audio capture and transcription.

- Capture: PulseAudio monitor source via ffmpeg subprocess
- Threading model: Capture → VAD → Worker → Display (4 threads)
- VAD: silero-vad v6, 512-sample frames (32ms at 16kHz), 10-frame speech onset / 20-frame silence offset
- Worker: faster-whisper transcribes speech segments in background
- Display: rich terminal live view + auto-appending .srt file
- Latency: ~3-6s total (buffer + VAD + whisper)
- CLI: `python -m src listen [-o output] [--model_size]`

Key files:
- `src/audio/capture.py` — PulseAudio monitor capture (Linux only)
- `src/transcription/stream.py` — streaming VAD + threaded pipeline
- `src/transcription/display.py` — rich terminal + SRT output
