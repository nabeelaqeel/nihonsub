# v0.2 — Windows + Cross-Platform Support

**Tag:** `v0.2`
**Branch:** `feat/windows-support` (merged to `main`)
**Date:** 2026-06-17

## Summary

Cross-platform live capture support. Windows users can now run live listen, with automatic fallback when ffmpeg lacks WASAPI input. macOS AVFoundation support added. VoiceMeeter virtual audio device detection for Bluetooth speaker setups.

## Added

### Windows Live Listen
Real-time system audio capture on Windows with dual-engine fallback.

- **Primary engine**: ffmpeg WASAPI loopback (lowest latency)
- **Fallback engine**: sounddevice via PortAudio (works even when ffmpeg lacks WASAPI input support)
- Device auto-detection: tries WASAPI first, falls back to sounddevice
- sounddevice device selection prioritizes: VoiceMeeter → VAIO → CABLE → loopback → Stereo Mix → any input device
- Detailed engine selection: see `docs/features/windows-support.md`

Key files:
- `src/audio/capture.py` — dual engine dispatch (`_capture_config()`)
- `pyproject.toml` — added `sounddevice>=0.5.0`

### macOS Live Listen
- AVFoundation audio capture via ffmpeg
- BlackHole virtual audio device auto-detection
- Same threaded pipeline as Linux/Windows after capture

### VoiceMeeter Detection
- `_find_sounddevice_device()` now recognizes: "voicemeeter", "voice meeter", "vaio", "cable", "loopback", "stereo mix" as priority devices
- Returns `(index, name)` tuple instead of just index for better diagnostics

### Audio Level Meter
- Real-time RMS audio level displayed in live listen header
- Visual bar: `Audio: [██████░░░░] 50%`
- Shows audio is flowing even before VAD triggers transcription
- Source: `stream.current_rms` updated on each audio chunk

Key files:
- `src/transcription/stream.py` — `current_rms` attribute
- `src/transcription/display.py` — `_audio_meter()` method
- `src/__main__.py` — wires RMS to display each frame

### Debug Visibility
- Capture engine print: `Capture engine: sounddevice (device 2)`
- Device name print: `Using sounddevice capture device: VoiceMeeter Output (index 2)`
- Audio level meter in live display (see above)

### VoiceMeeter Setup Guide
- `docs/voicemeeter.md` — complete guide covering:
  - Why VoiceMeeter (hear + capture simultaneously with Bluetooth speakers)
  - Installation steps
  - Routing configuration (Virtual Input → A1 speakers + B1 virtual output)
  - nihonsub auto-detection
  - Troubleshooting
